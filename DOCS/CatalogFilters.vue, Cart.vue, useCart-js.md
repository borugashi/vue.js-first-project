Это три ключевых файла для понимания того, как во Vue работает **состояние (state)**, **переиспользуемая логика (composables)** и **двустороннее связывание (v-model)**.

Начнем с фундамента — с файла `useCart.js`, так как он используется и в корзине, и в карточке товара.

### 1. useCart.js (Логика корзины / Composable)

Этот файл — не компонент Vue (тут нет `<template>`), а **Composable** (композабл).

- **Что такое Composable:** Это обычная JavaScript-функция, которая использует Composition API (`ref`, `computed`, `watch`) и возвращает данные и функции.
- **Зачем нужен:** Чтобы одну и ту же логику (добавление в корзину, подсчет суммы) можно было использовать в разных компонентах (`Header.vue`, `ProductCard.vue`, `Cart.vue`), не дублируя код. Это аналог миксинов или хуков в React.

#### А. Работа с `localStorage` и JSON

```javascript
const CART_KEY = "muscle-cart";
const loadCart = () => {
  try {
    const saved = localStorage.getItem(CART_KEY);
    return saved ? JSON.parse(saved) : [];
  } catch {
    return [];
  }
};
```

- **`localStorage`**: Это встроенное в браузер хранилище данных. В отличие от переменных в JS, данные в `localStorage` **не исчезают** при перезагрузке страницы или закрытии вкладки. Они хранятся на компьютере пользователя.
- **Проблема `localStorage`**: Он умеет хранить только **строки**. Если мы сохраним туда объект или массив, он превратится в строку `"[object Object]"`.
- **`JSON.stringify()` и `JSON.parse()`**:
  - `stringify` (строкаify) — превращает JS-объект/массив в строку в формате JSON (чтобы сохранить).
  - `parse` (парсить) — превращает строку JSON обратно в JS-объект/массив (чтобы прочитать).
- **`try/catch`**: Если в хранилище лежит "битая" строка, `JSON.parse` вызовет ошибку. `try/catch` ловит её и возвращает пустой массив `[]`, чтобы сайт не упал.

#### Б. Реактивность и `watch`

```javascript
const cartItems = ref(loadCart());

watch(
  cartItems,
  (newCart) => {
    localStorage.setItem(CART_KEY, JSON.stringify(newCart));
  },
  { deep: true },
);
```

- **`ref(loadCart())`**: При запуске сайта мы сразу читаем корзину из памяти браузера и делаем её реактивной.
- **`watch`**: Мы говорим Vue: «Следи за `cartItems`. Как только массив изменится (добавили товар, удалили, поменяли количество) — выполни функцию».
- **`{ deep: true }`**: Обязательно для массивов объектов. Если мы просто изменим свойство `quantity` у одного из товаров внутри массива, обычный `watch` этого не заметит. `deep` заставляет Vue проверять вложенные изменения.

#### В. Вычисляемые свойства (`computed`)

```javascript
const totalCount = computed(() => cartItems.value.reduce((sum, item) => sum + item.quantity, 0));
const totalPrice = computed(() => cartItems.value.reduce((sum, item) => sum + item.price * item.quantity, 0));
```

- **Что такое `computed`:** Это "умная" переменная. Она **автоматически пересчитывается** только тогда, когда меняются зависимые от неё реактивные данные (в данном случае `cartItems`).
- **Зачем не использовать обычную функцию?** `computed` **кеширует** результат. Если вы обратитесь к `totalCount` 100 раз в шаблоне, функция выполнится только 1 раз (пока массив не изменится). Обычная функция вызывалась бы 100 раз при каждой перерисовке.
- **Метод массива `reduce`**: Это "сворачивание" массива в одно значение.
  - `(sum, item) => ...` — функция, которая выполняется для каждого элемента.
  - `sum` — аккумулятор (накопительная сумма). В начале он равен `0` (второй аргумент `reduce`).
  - Для каждого `item` мы прибавляем его количество (или цену \* количество) к `sum` и возвращаем новое значение `sum`.

#### Г. Методы массивов в функциях

- **`find`**: Ищет **первый** элемент, удовлетворяющий условию, и возвращает его. Если не нашел — `undefined`. Используется в `addToCart` (проверить, есть ли товар) и `updateQuantity`.
- **`filter`**: Создает **новый** массив, оставляя только те элементы, для которых условие истинно. Используется в `removeFromCart`: `item.id !== productId` (оставить всё, кроме удаляемого).

### 2. Cart.vue (Страница корзины)

Этот файл отрисовывает интерфейс, используя логику из `useCart.js`.

#### А. Деструктуризация composable

```javascript
const { cartItems, removeFromCart, updateQuantity, clearCart, totalPrice } = useCart();
```

- Мы вызываем функцию `useCart()` и сразу "вытаскиваем" (деструктурируем) из возвращаемого объекта только те переменные и функции, которые нужны именно этой странице.

#### Б. Вычисления прямо в шаблоне

```html
<p class="font-bold text-gray-900 w-24 text-center">{{ item.price * item.quantity }} ₽</p>
```

- Внутри `{{ }}` можно писать любые простые JS-выражения. Умножение цены на количество — идеальный пример. Но писать там сложные условия или циклы нельзя (для этого есть `v-if` и `v-for`).

#### В. Адаптивная верстка (Flexbox)

```html
<div class="bg-white ... flex flex-col sm:flex-row items-center gap-4"></div>
```

- **`flex-col sm:flex-row`**: Классический паттерн адаптивности. На мобильных (`flex-col`) картинка, название и кнопки идут друг под другом (в колонку). На экранах от `sm` (640px) они выстраиваются в строку (`flex-row`).
- **`flex-1`**: У блока с информацией о товаре. Это заставляет блок занять всё свободное пространство во флексе, прижимая кнопки количества и удаления к правому краю.

### 3. CatalogFilters.vue (Компонент фильтров)

Здесь мы видим, как Vue работает с формами и как компоненты общаются друг с другом.

#### А. Двустороннее связывание (`v-model`) на разных элементах

- **`<input type="radio" v-model="selectedCategory">`**: `v-model` автоматически связывает выбранный radio-кнопку с переменной. Если `selectedCategory` равна `id` категории, эта кнопка будет отмечена.
- **`<input type="checkbox" v-model="isNew">`**: Для чекбокса `v-model` работает с булевыми значениями (`true`/`false`).
- **`<input type="number" v-model.number="priceMin">`**: Модификатор `.number` важен! По умолчанию любой инпут возвращает **строку** (даже если `type="number"`). `.number` автоматически превращает ввод в число, чтобы мы могли корректно сравнивать цены.

#### Б. События инпутов: `@input` vs `@change`

```html
<input type="number" v-model.number="priceMin" @input="$emit('filter-change', ...)" />
```

- **`@input`**: Срабатывает **мгновенно** при каждом нажатии клавиши (вводе каждого символа). Для фильтрации по цене это удобно — пользователь видит результат сразу.
- **`@change`**: Срабатывает, когда элемент теряет фокус (кликнули в другое место) или изменил значение (для select/checkbox). В этом файле `@change` используется для радио-кнопок и чекбоксов, так как там нет смысла отправлять запрос при каждом клике, достаточно факта изменения.

#### В. `defineEmits` и общение с родителем

```javascript
const emit = defineEmits(["filter-change"]);
// ...
$emit("filter-change", { categoryId: selectedCategory });
```

- **`defineEmits`**: Объявляет "контракт" компонента. Мы говорим Vue: "Этот компонент может отправлять событие с именем `filter-change`".
- **`$emit`**: Метод, который фактически отправляет событие наверх, родителю (`Catalog.vue`). Вместе с событием мы передаем **полезную нагрузку** (payload) — объект с новыми значениями фильтров.
- _Почему не менять пропсы напрямую?_ Напомню правило: пропсы менять нельзя. Ребенок не знает, как именно родитель хранит данные. Он просто кричит: "Эй, тут поменяли категорию на 5!", а родитель сам решает, что с этим делать.

#### Г. `useRoute` и чтение URL

```javascript
import { useRoute } from "vue-router";
const route = useRoute();
// ...
if (route.query.category) { ... }
```

- **`useRoute`**: Хук Vue Router, который дает доступ к информации о текущем маршруте.
- **`route.query`**: Это объект с GET-параметрами из адресной строки. Если пользователь перешел по ссылке `/catalog?category=1` (например, кликнув на "Протеин" в `PopularCategories.vue`), то `route.query.category` будет равен `"1"`.
- **Логика в `onMounted`**: При загрузке страницы фильтров мы проверяем URL. Если там есть категория, мы сразу устанавливаем её в `selectedCategory` и отправляем событие родителю, чтобы товары отфильтровались без лишних кликов.

### Шпаргалка для ответа на экзамене

**(Vue, код, логика):**

- **В:** «Что такое `computed` и чем отличается от обычной функции или `ref`?»
  **О:** «`computed` — это вычисляемое свойство с кешем. Оно пересчитывается только когда меняются его реактивные зависимости. Обычная функция в шаблоне вызывалась бы при каждом рендере, что хуже для производительности. От `ref` отличается тем, что `computed` нельзя изменить напрямую (у него нет сеттера по умолчанию), оно только читается.»
- **В:** «Зачем в `watch` для корзины вы используете `{ deep: true }`?»
  **О:** «Потому что корзина — это массив объектов. Если я просто изменю `quantity` у одного товара, ссылка на сам массив не изменится. `deep: true` заставляет Vue следить за вложенными свойствами и запускать сохранение в `localStorage`.»
- **В:** «Как вы передаете данные из фильтров в каталог?»
  **О:** «Через кастомные события. В `CatalogFilters` я объявляю `defineEmits(['filter-change'])`. При изменении инпута вызываю `$emit` и передаю объект с новыми данными. Родитель `Catalog.vue` слушает это событие через `@filter-change` и обновляет свой `ref` с фильтрами, который потом прокидывает в `CatalogProducts`.»
- **В:** «Зачем в инпуте цены модификатор `.number` у `v-model`?»
  **О:** «HTML-инпуты всегда возвращают строку. Чтобы корректно сравнивать цены (например, `priceMin < 1000`), мне нужно число. Модификатор `.number` автоматически кастит ввод в тип Number.»

**(Теория JS, Браузер, Архитектура):**

- **В:** «Что такое `localStorage`? Какие у него ограничения?»
  **О:** «Это клиентское хранилище браузера (Web Storage API). Данные хранятся в виде пар ключ-значение (только строки) и сохраняются между сессиями (не удаляются при закрытии вкладки). Ограничения: обычно около 5-10 МБ на домен, данные не отправляются на сервер автоматически (в отличие от куки), и они доступны только в рамках одного домена/протокола.»
- **В:** «Что такое JSON и зачем нужны `parse` и `stringify`?»
  **О:** «JSON (JavaScript Object Notation) — это текстовый формат обмена данными. `localStorage` работает только со строками. `JSON.stringify` сериализует (превращает в строку) JS-объект для сохранения, а `JSON.parse` десериализует (превращает обратно в объект) строку при чтении.»
- **В:** «Объясните, как работает метод массива `reduce`.»
  **О:** «`reduce` "сворачивает" массив в одно значение. Он принимает функцию-колбэк и начальное значение аккумулятора. Колбэк выполняется для каждого элемента, возвращая новое значение аккумулятора, которое передается в следующую итерацию. В конце возвращается итоговое значение.»
- **В:** «В чем разница между `@input` и `@change`?»
  **О:** «Событие `input` срабатывает мгновенно при любом изменении значения элемента (например, при вводе каждого символа в текстовое поле). Событие `change` срабатывает, когда изменение зафиксировано: элемент потерял фокус (blur) или было выбрано новое значение в выпадающем списке/чекбоксе.»
