### Header.vue (Шапка сайта)

**1. Что такое компонент и структура файла**
Любой файл `.vue` в Vue 3 состоит из трех блоков (хотя некоторые могут отсутствовать):

- `<template>` — HTML-разметка (то, что видит пользователь).
- `<script setup>` — JavaScript-логика (данные, функции, импорты). Приставка `setup` означает, что мы используем современный синтаксис Composition API (код выполняется один раз при создании компонента).
- `<style>` — CSS-стили (в вашем проекте стили вынесены в Tailwind, поэтому этого блока нет).

**2. Интерполяция `{{ totalCount }}`**
В тексте шапки вы видите `{{ totalCount }}`.

- **Что это:** Двойные фигурные скобки во Vue называются **интерполяцией** (или Mustache-синтаксисом). Они позволяют вставить JavaScript-переменную прямо в HTML.
- **Откуда берется `totalCount`:** Скорее всего, это количество товаров в корзине. Во Vue данные должны быть **реактивными** (то есть если переменная меняется в коде, она автоматически меняется на экране). Обычно это делается через `ref` или `computed` (о них поговорим позже), либо `totalCount` передается сюда через **пропсы**.
- **Что такое пропсы (props):** Пропсы — это свойства, которые родительский компонент передает дочернему. _Аналогия из JS:_ если компонент — это функция, то пропсы — это её аргументы. Если `Header` получает `totalCount` от `App.vue`, то в `Header` это объявляется через `defineProps(['totalCount'])`.

**3. Навигация и `<router-link>`**
Вместо обычных тегов `<a>` для меню используются `<router-link>`.

- **Зачем:** Ваш сайт — это SPA (Single Page Application). Обычный тег `<a>` заставляет браузер перезагрузить страницу целиком. `<router-link>` перехватывает клик и меняет только нужную часть страницы (компонент внутри `<router-view>`) без перезагрузки, используя Vue Router.

**4. Tailwind CSS в Header**

- `flex` — включает Flexbox (модель для выравнивания элементов в строку или колонку).
- `gap-...` — задает отступы между элементами внутри флекса.
- `hover:text-[#196950]` — меняет цвет текста при наведении мыши. Квадратные скобки `[]` в Tailwind позволяют использовать любые кастомные значения (в данном случае конкретный HEX-код цвета).
- `transition` — делает смену цвета плавной, а не мгновенной.

### Footer.vue (Подвал сайта)

**1. Пустой `<script setup></script>`**
В конце файла вы видите пустой скрипт.

- **Зачем он нужен:** Во Vue 3, чтобы файл воспринимался как полноценный компонент (особенно при использовании `<script setup>`), этот тег должен присутствовать, даже если в компоненте нет никакой JS-логики (как в футере — там только верстка).

**2. Разметка и адаптивность (Tailwind)**
Футер содержит сложную верстку, которая перестраивается на мобильных устройствах.

- `max-w-7xl mx-auto` — ограничивает максимальную ширину контента (чтобы на огромных мониторах сайт не растягивался на весь экран) и центрирует его (`mx-auto` = margin left/right: auto).
- `flex-col md:flex-row` — **ключевой момент для адаптивности**. Tailwind использует подход _Mobile First_. `flex-col` означает, что по умолчанию (на мобильных) элементы выстраиваются в колонку. Префикс `md:` означает "на экранах от 768px и выше применить `flex-row`" (выстроить в строку).
- `space-y-2` — добавляет вертикальные отступы между всеми дочерними элементами внутри блока (аналог `gap`, но для обычных блоков, не только флексов).
- `py-8 md:py-12` — `p` это padding (внутренний отступ), `y` означает "по вертикали" (сверху и снизу). На мобильных отступ 8, на планшетах/ПК — 12.

**3. Ссылки: `<router-link>` vs `<a>`**

- Для внутренних страниц ("О нас", "Контакты") используется `<router-link to="/...">`, чтобы не перезагружать SPA.
- Для соцсетей (TG, WhatsApp, VK) используется обычный `<a href="#">`. Почему? Потому что это внешние ссылки (или заглушки, так как там `#`), они не обрабатываются Vue Router.

### Шпаргалка для ответа на экзамене

**(по коду и Vue):**

- **Вопрос:** "Почему в меню вы использовали `router-link`, а для иконок соцсетей обычный `a`?"
  **Ответ:** "`router-link` используется для внутренних переходов, чтобы Vue Router мог менять компоненты без перезагрузки страницы (SPA). Соцсети — это внешние ссылки, поэтому там стандартный тег `a`."
- **Вопрос:** "Как работает `{{ totalCount }}` в шапке? Откуда оно берется?"
  **Ответ:** "Это интерполяция Vue. Переменная `totalCount` реактивна. Она либо импортируется из composable-функции (например, `useCart`), либо передается через пропсы от родительского компонента. При изменении данных в хранилище, Vue автоматически обновляет это число в DOM."
- **Вопрос:** "Что делает `md:flex-row`?"
  **Ответ:** "Это утилита Tailwind CSS для адаптивной верстки. По умолчанию элементы идут колонкой (для мобильных), а при достижении брейкпоинта `md` (768px) они выстраиваются в строку."

**(по теории):**

- **Вопрос:** "Чем ваш подход к переходам между страницами отличается от классического сайта?"
  **Ответ:** "Классический сайт (MPA) при клике на ссылку отправляет запрос на сервер, получает новый HTML и полностью перерисовывает страницу. Мой сайт — это SPA (Single Page Application). Сервер один раз отдает пустой HTML и JS-бандл. Дальше переходы между 'страницами' происходят на клиенте: Vue Router просто подменяет компоненты внутри `<router-view>`, что делает интерфейс мгновенным."
