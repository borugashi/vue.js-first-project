Разбираем три важных файла: **`ProductCard.vue`**, **`NewProducts.vue`** и **`PopularCategories.vue`**. Здесь начинается настоящая магия Vue: передача данных, загрузка с сервера, циклы и условия.

---

### 1. ProductCard.vue (Карточка товара)

Это переиспользуемый компонент. Он не знает, какой именно товар он отображает, он просто ждет, что родительский компонент передаст ему данные.

#### А. Пропсы (Props) — связь «Родитель → Ребенок»

```javascript
const props = defineProps({
  product: Object,
});
```

- **Что такое пропсы:** Это аргументы, которые родительский компонент передает дочернему. _Аналогия из JS:_ если компонент — это функция `ProductCard(product)`, то пропсы — это параметр `product`.
- **`defineProps`:** Макрос Vue 3 (в `<script setup>`), который объявляет, какие данные компонент ожидает получить извне.
- **`product: Object`:** Мы говорим Vue: «Ждем пропс с именем `product`, и он должен быть объектом». Это базовая валидация.
- **Важное правило:** Пропсы во Vue **однонаправленные**. Родитель может менять пропсы, а ребенок — нет (нельзя делать `props.product.name = 'New Name'`). Если ребенку нужно свое изменяемое состояние на основе пропса, он должен создать локальную переменную через `ref`.

#### Б. Динамические атрибуты и шаблонные строки

```html
<router-link :to="`/products/${product.id}`">
  <img :src="product.image || '/imgs/placeholder-product.png'" :alt="product.name" />
</router-link>
```

- **Двоеточие (`:`):** Это сокращение от директивы `v-bind`. Оно говорит Vue: «Не воспринимай это как обычный текст, выполни JavaScript-выражение внутри кавычек».
- **Обратные кавычки (`` ` ``) и `${}`:** Это **шаблонные строки** (template literals) из ES6. Они позволяют вставлять переменные прямо в строку. Мы склеиваем путь `/products/` и `id` товара, чтобы получить ссылку вида `/products/15`.
- **Логическое ИЛИ (`||`):** `product.image || '/imgs/placeholder...'`. Если у товара нет картинки (`product.image` равно `null` или `undefined`), JS вернет второе значение (картинку-заглушку). Это классический паттерн **fallback** (резервный вариант).

#### В. События и модификаторы

```html
<button @click.prevent.stop="handleAddToCart"></button>
```

- **`@click`:** Сокращение от `v-on:click`. Мы «слушаем» событие клика по кнопке.
- **Модификаторы (точки после события):**
  - `.prevent`: Вызывает `event.preventDefault()`. Отменяет стандартное поведение браузера (например, если бы это была форма, она бы не перезагрузила страницу).
  - `.stop`: Вызывает `event.stopPropagation()`. **Критически важно здесь!** Кнопка находится внутри `<router-link>`. По умолчанию клик по кнопке «всплывает» (bubble up) к родителю-ссылке, и происходит переход на страницу товара. `.stop` останавливает всплытие: клик обрабатывается кнопкой (добавляет в корзину), но переход по ссылке не происходит.

#### Г. Composable (Композабл) `useCart`

```javascript
import { useCart } from "@/composables/useCart";
const { addToCart } = useCart();
```

- **Что это:** Функция, которая возвращает логику корзины (скорее всего, там внутри `ref` с массивом товаров и функция добавления). Мы импортируем её и используем деструктуризацию объекта (`const { addToCart }`), чтобы сразу получить нужную функцию.

### 2. NewProducts.vue (Секция «Новинки»)

Здесь мы загружаем данные с сервера. Это классический пример работы с асинхронностью и реактивностью.

#### А. Реактивность через `ref`

```javascript
import { ref, onMounted } from "vue";
const products = ref([]);
const loading = ref(false);
const error = ref(null);
```

- **Что такое `ref`:** Функция Vue, которая делает переменную **реактивной**. Vue создает прокси-объект вокруг значения. Когда значение меняется, Vue автоматически перерисовывает шаблон.
- **Правило `.value`:**
  - В `<script>`: Чтобы прочитать или изменить значение, нужно писать `products.value = [...]`.
  - В `<template>`: Vue сам «распаковывает» ref, поэтому пишем просто `{{ products }}` или `v-for="product in products"`.

#### Б. Хук жизненного цикла `onMounted`

```javascript
onMounted(async () => { ... });
```

- **Что такое хуки жизненного цикла:** Компонент во Vue проходит этапы: создание, монтирование в DOM, обновление, уничтожение.
- **`onMounted`:** Срабатывает один раз, когда компонент уже вставлен в HTML-страницу (DOM). Запросы к API обычно делают здесь, чтобы не блокировать отрисовку интерфейса.

#### В. Асинхронность (`async/await` и `try/catch`)

```javascript
onMounted(async () => {
  loading.value = true;
  try {
    const response = await getProducts({ is_new: true, per_page: 4 });
    products.value = response.data.data || [];
  } catch (err) {
    error.value = "Ошибка загрузки новинок";
  } finally {
    loading.value = false;
  }
});
```

- **`async/await`:** Современный синтаксис работы с **Промисами (Promises)**.
  - _Что такое Promise:_ Объект, представляющий будущее значение. Запрос к серверу занимает время. `await` говорит JS: «Подожди, пока этот запрос не завершится (не вернет результат или ошибку), и только потом иди дальше».
- **`try/catch/finally`:** Блок обработки ошибок.
  - `try`: Пытаемся выполнить код.
  - `catch`: Если в `try` произошла ошибка (сервер упал, нет интернета), попадаем сюда. Сохраняем текст ошибки в `error.value`, чтобы показать его пользователю.
  - `finally`: Выполняется в любом случае (была ошибка или нет). Здесь мы выключаем индикатор загрузки (`loading.value = false`).

#### Г. Условный рендеринг (`v-if`, `v-else-if`, `v-else`)

```html
<div v-if="loading">Загрузка...</div>
<div v-else-if="error">{{ error }}</div>
<div v-else class="grid ...">...карточки...</div>
```

- **Как работает:** Vue проверяет условия по порядку. Если `loading` истинно — показывает первый блок, остальные даже не создаются в DOM. Если нет — проверяет `error`. Если и его нет — показывает сетку с товарами. Это эффективнее, чем скрывать блоки через CSS (`display: none`), так как элементы физически не существуют в памяти, пока не нужны.

#### Д. Рендеринг списков (`v-for` и `:key`)

```html
<ProductCard v-for="product in products" :key="product.id" :product="product" />
```

- **`v-for="product in products"`:** Цикл. Для каждого элемента массива `products` создается копия компонента `ProductCard`, и текущий элемент сохраняется в переменную `product`.
- **`:key="product.id"`:** **Обязательное требование Vue при использовании `v-for`**.
  - _Зачем:_ Vue использует Виртуальный DOM. Когда массив меняется (например, добавился товар), Vue сравнивает старый и новый список. Уникальный `key` помогает ему понять, какой именно компонент обновить, переместить или удалить, не перерисовывая весь список целиком. Это сильно ускоряет работу.

### 3. PopularCategories.vue (Популярные категории)

Здесь нет JS-логики, но есть отличная верстка на **CSS Grid** и работа с параметрами URL.

#### А. CSS Grid vs Flexbox

```html
<div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-4 gap-6"></div>
```

- **В чем разница:**
  - **Flexbox** (одномерный): Выстраивает элементы в строку ИЛИ в колонку. Идеален для меню, выравнивания элементов внутри строки.
  - **Grid** (двумерный): Работает сразу со строками и колонками. Идеален для сложных сеток (как здесь), где элементы должны выстраиваться в таблицу.
- **`grid-cols-1 md:grid-cols-2 lg:grid-cols-4`:** Адаптивная сетка. 1 колонка на мобильных, 2 на планшетах, 4 на десктопах.

#### Б. Объединение ячеек

```html
<router-link ... class="lg:col-span-2 ..."></router-link>
```

- **`col-span-2`:** Аналог `colspan` в таблицах HTML. Блок «Протеин» занимает две колонки из четырех на больших экранах, чтобы выделить его и дать место для списка преимуществ.

#### В. Query-параметры в URL

```html
<router-link to="/catalog?category=1"></router-link>
```

- **Path params vs Query params:**
  - В `router.js` у вас есть `/products/:id`. Это **path parameter** (часть пути). Он обязателен для маршрута.
  - Здесь используется `?category=1`. Это **query parameter** (параметр запроса). Он не обязателен, используется для фильтрации, сортировки, пагинации.
  - На странице каталога вы будете считывать этот параметр (через `useRoute().query.category`), чтобы показать товары только нужной категории.

### Шпаргалка для ответа на экзамене

**(Vue, код, логика):**

- **В:** «Зачем здесь `:key` в `v-for`?»
  **О:** «Vue использует виртуальный DOM. Уникальный ключ помогает ему отслеживать изменения в списке и обновлять только те компоненты, которые реально изменились, а не перерисовывать весь список. Это нужно для производительности и корректного сохранения состояния компонентов.»
- **В:** «Почему в скрипте вы пишете `products.value`, а в шаблоне просто `products`?»
  **О:** «`ref` оборачивает значение в реактивный объект. В JS-коде мы должны обращаться к свойству `.value`, чтобы получить или изменить данные. В шаблоне Vue автоматически распаковывает (unwraps) ref, поэтому `.value` писать не нужно.»
- **В:** «Что делает `.stop` в `@click.stop` на кнопке "В корзину"?»
  **О:** «Останавливает всплытие события (event bubbling). Кнопка находится внутри `router-link`. Без `.stop` клик по кнопке всплыл бы к ссылке, и произошел бы переход на страницу товара. Модификатор позволяет выполнить только функцию добавления в корзину.»
- **В:** «Как работает `async/await` и зачем тут `try/catch`?»
  **О:** `await` приостанавливает выполнение функции, пока промис (запрос к API) не разрешится. `try/catch` нужен для обработки ошибок: если сервер вернет 500 или пропадет интернет, мы не сломаем приложение, а покажем пользователю сообщение об ошибке через переменную `error`.

**(Теория JS, HTTP, Веб):**

- **В:** «Что такое Promise и чем `async/await` лучше старых колбэков?»
  **О:** «Promise — это объект, представляющий результат асинхронной операции (успех или ошибку). Раньше использовали колбэк-функции, что приводило к "аду колбэков" (вложенность). `async/await` — это синтаксический сахар над промисами, который позволяет писать асинхронный код так, будто он синхронный, что сильно улучшает читаемость.»
- **В:** «Чем GET-параметры (query) отличаются от параметров пути (path)?»
  **О:** «Path-параметры (например, `/products/15`) — это часть URL, они идентифицируют конкретный ресурс и обычно обязательны. Query-параметры (например, `?category=1&sort=price`) идут после знака `?`, они не обязательны и используются для фильтрации, сортировки или пагинации коллекции ресурсов.»
- **В:** «В чем разница между Flexbox и CSS Grid?»
  **О:** «Flexbox — одномерная система (строка ИЛИ колонка), она лучше подходит для выравнивания элементов внутри контейнера и распределения свободного пространства. Grid — двумерная система (строки И колонки одновременно), она идеальна для создания сложных макетов и сеток, где элементы должны занимать несколько ячеек.»
- **В:** «Что такое однонаправленный поток данных (пропсы)?»
  **О:** «Во Vue данные передаются от родителя к ребенку через пропсы. Ребенок не может напрямую менять пропсы родителя. Это нужно для того, чтобы состояние приложения было предсказуемым: если данные изменились, мы всегда знаем, какой именно компонент их изменил (родитель), а не ищем баг по всем дочерним компонентам.»
