### 1. ReviewCard.vue (Карточка отзыва)

Это «глупый» (презентационный) компонент. Он просто получает данные и красиво их рисует.

#### А. Условный рендеринг (`v-if` / `v-else`)

```html
<img v-if="avatarUrl" :src="avatarUrl" ... />
<div v-else class="...">
  <span>{{ review.name.charAt(0) }}</span>
</div>
```

- **Как работает:** Если пропс `avatarUrl` передан и он не пустой, Vue отрисует тег `<img>`. Если нет (например, у пользователя нет фото) — отрисует блок `v-else` с круглым фоном и первой буквой имени.
- **Метод строки `charAt(0)`:** Возвращает символ по индексу. Индексы в JS начинаются с 0, поэтому `charAt(0)` берет первую букву.
- **Важно:** `v-if` и `v-else` должны идти **строго подряд**, без других элементов между ними, иначе Vue потеряет связь и выдаст ошибку.

#### Б. Динамические стили и тернарный оператор

```html
<svg v-for="star in 5" :key="star" :fill="star <= review.rating ? '#196950' : '#E5E7EB'"></svg>
```

- **`:fill`**: Динамическая привязка атрибута `fill` (цвет заливки SVG).
- **Тернарный оператор (`? :`)**: Это сокращенная форма `if/else` в одну строку.
  - _Синтаксис:_ `условие ? значение_если_истина : значение_если_ложь`.
  - _Логика:_ Если номер текущей звезды (`star`) меньше или равен рейтингу отзыва (`review.rating`), закрашиваем её зеленым (`#196950`), иначе — серым (`#E5E7EB`).

### 2. ReviewsSection.vue (Секция отзывов на главной)

Здесь мы подключаем стороннюю библиотеку **Swiper** (популярный слайдер/карусель).

#### А. Интеграция сторонних Vue-компонентов

```javascript
import { Swiper, SwiperSlide } from "swiper/vue";
import { Pagination } from "swiper/modules";
```

- **Что такое npm-пакеты:** Swiper устанавливается через `npm install swiper`. Это готовый код, который мы импортируем в проект.
- **Передача сложных данных в пропсы:**
  ```html
  <Swiper :modules="modules" :breakpoints="{ 768: { slidesPerView: 2 } }"></Swiper>
  ```
  Обратите внимание на двоеточие перед `modules` и `breakpoints`. Если его не поставить, Swiper воспримет это как обычную строку. Двоеточие (`v-bind`) говорит Vue: «Выполни это как JavaScript и передай массив/объект».

#### Б. Маппинг (Словарь) данных

```javascript
const avatarMap = { 1: "/imgs/review-1.jpg", 2: "/imgs/review-2.jpg" ... };
const getAvatarUrl = (reviewId) => avatarMap[reviewId] || "/imgs/review-1.jpg";
```

- **Зачем:** Бэкенд возвращает отзывы с ID (1, 2, 3...), но не возвращает ссылки на картинки (так как это учебный проект). Мы создаем JS-объект `avatarMap`, где ключи — это ID, а значения — пути к картинкам.
- **Обращение по ключу:** `avatarMap[reviewId]` — это динамическое получение значения из объекта по переменной. Если ID нет в словаре, срабатывает `||` (ИЛИ) и возвращается картинка по умолчанию.

### 3. ProductReviews.vue (Отзывы на странице товара)

Этот файл почти идентичен `ReviewsSection.vue`, но есть одно **критически важное отличие**, связанное с Vue Router.

#### А. Наблюдатель за пропсом (`watch`)

```javascript
watch(
  () => props.productId,
  () => {
    currentPage.value = 1;
    loadReviews();
  },
);
```

- **Проблема SPA:** Когда вы кликаете на товар в каталоге, Vue Router меняет URL (например, с `/products/1` на `/products/2`). Но страница (`ProductPage.vue`) **не перезагружается** целиком! Vue просто обновляет пропс `productId` внутри компонента `ProductInfo` и `ProductReviews`.
- **Почему не работает `onMounted`:** Хук `onMounted` срабатывает только **один раз** при первом появлении компонента. При переходе между товарами компонент не уничтожается, поэтому `onMounted` не вызовется, и отзывы от первого товара останутся висеть на втором.
- **Решение:** Мы используем `watch`, чтобы следить за изменением `productId`. Как только ID меняется, мы сбрасываем страницу на 1 и делаем новый запрос к API.

### 4. ContactsSection.vue (Контакты и Карта)

Статичный компонент с версткой и встраиванием внешней карты.

#### А. Тег `<iframe>`

```html
<iframe src="https://yandex.ru/map-widget/..." width="100%" height="100%" frameborder="0"></iframe>
```

- **Что это:** Inline Frame (встроенный фрейм). Позволяет вставить одну HTML-страницу внутрь другой. В данном случае мы вставляем виджет Яндекс.Карт.
- **Атрибуты:**
  - `src`: Ссылка на встраиваемый ресурс.
  - `width/height="100%"`: Растягивает карту на весь размер родительского блока.
  - `frameborder="0"`: Убирает стандартную рамку вокруг фрейма (устаревший атрибут, но иногда используется для совместимости, в CSS это делается через `border: none`).
- **Стилизация контейнера:** Сам `<iframe>` находится внутри `div` с классами `rounded-2xl overflow-hidden border-3`.
  - `overflow-hidden` здесь обязателен! Если его не будет, острые углы карты "вылезут" за скругленные углы (`rounded-2xl`) родительского блока.

### Шпаргалка для ответа на экзамене

**(Vue, код, логика):**

- **В:** «Как вы сделали звездочки рейтинга? Почему они разного цвета?»
  **О:** «Я использовал цикл `v-for="star in 5"`, чтобы создать 5 SVG-иконок. Цвет заливки (`fill`) задается динамически через тернарный оператор: если номер звезды меньше или равен рейтингу из данных, она зеленая, иначе серая.»
- **В:** «Зачем в `ProductReviews` вы используете `watch` для `productId`, если у вас уже есть `onMounted`?»
  **О:** «Потому что это SPA. При переходе со страницы одного товара на другой компонент не пересоздается, а просто обновляются пропсы. `onMounted` сработает только один раз при первом заходе. `watch` же отслеживает изменение ID и перезагружает отзывы для нового товара.»
- **В:** «Что такое `avatarMap` и как вы его используете?»
  **О:** «Это обычный JS-объект, который работает как словарь. Ключи — это ID отзывов, значения — пути к картинкам. Я передаю ID в функцию и получаю нужный URL через квадратные скобки `avatarMap[id]`.»

**(Теория JS, Браузер, Веб):**

- **В:** «Что такое тернарный оператор? Приведите пример.»
  **О:** «Это оператор условия, который возвращает одно из двух значений в зависимости от истинности условия. Синтаксис: `условие ? значение1 : значение2`. Аналог конструкции `if/else`, но используется для краткой записи, часто внутри шаблонов или присваиваний.»
- **В:** «Что такое `<iframe>` и зачем он нужен? Есть ли у него минусы?»
  **О:** `iframe` позволяет встроить внешний документ (например, карту или видео) прямо в страницу. Минусы: он может замедлять загрузку страницы, так как браузеру нужно грузить отдельный документ; могут быть проблемы с адаптивностью (нужно правильно задавать ширину/высоту); и поисковики плохо индексируют содержимое фреймов.
- **В:** «В чем разница между `v-if` и `v-show`?» _(Классический вопрос, который часто задают в контексте `v-if` из ReviewCard)_
  **О:** `v-if` полностью удаляет элемент из DOM, если условие ложно, и создает заново, если истинно. Это требует больше ресурсов при переключении. `v-show` просто переключает CSS-свойство `display: none`. Элемент всегда остается в DOM. `v-if` лучше для редких переключений, `v-show` — для частых (например, скрытие/показ меню).
