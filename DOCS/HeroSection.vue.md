### 1. Наложение текста на картинку (CSS Positioning)

Самая сложная часть для новичков — сделать так, чтобы текст был _поверх_ картинки, а картинка при этом растягивалась на весь блок.

```html
<section class="relative w-full h-[600px] md:h-[900px] overflow-hidden">
  <img src="..." class="absolute inset-0 w-full h-full object-cover z-0" />
  <div class="relative z-10 ...">
    <!-- Текст -->
  </div>
</section>
```

**Как это работает (разбор по полочкам):**

1.  **`relative` (у `<section>`)**: Мы говорим браузеру: «Этот блок будет точкой отсчета для всего, что внутри него позиционируется абсолютно». Если этого не сделать, картинка улетит в угол всего экрана.
2.  **`absolute` (у `<img>`)**: Картинка «вырывается» из обычного потока документа. Она перестает влиять на высоту и ширину родителя.
3.  **`inset-0`**: Это удобная утилита Tailwind, которая заменяет сразу четыре свойства: `top: 0; right: 0; bottom: 0; left: 0;`. Она растягивает картинку на весь родительский блок.
4.  **`z-0` и `z-10` (Z-index)**: Представьте, что экран — это ось Z (глубина). По умолчанию все элементы находятся на слое 0. Картинке мы явно задали `z-0`, а блоку с текстом — `z-10`. Чем больше число, тем «ближе» элемент к пользователю. Поэтому текст лежит поверх картинки.
5.  **`overflow-hidden`**: Обрезает всё, что выходит за границы `<section>`. Нужно, чтобы картинка не вылезала за скругленные углы (если бы они были) или за заданную высоту.

### 2. Адаптивность и подход Mobile First

Ваш сайт должен хорошо выглядеть и на телефоне, и на большом мониторе. Tailwind использует подход **Mobile First** (сначала стили для мобильных, потом для больших экранов).

- **`h-[600px] md:h-[900px]`**:
  - `h-[600px]` — высота 600 пикселей для мобильных.
  - `md:h-[900px]` — когда экран становится шире брейкпоинта `md` (768px, планшеты и ПК), высота меняется на 900px.
  - **Квадратные скобки `[]`**: В Tailwind есть стандартные размеры (например, `h-96`), но если вам нужен точный размер, вы пишете его в скобках. Tailwind сгенерирует обычный CSS `height: 600px;`.
- **`text-2xl md:text-4xl lg:text-5xl`**: Размер шрифта. На телефоне он маленький (`2xl`), на планшете средний (`4xl`), на больших экранах (`lg`, от 1024px) — огромный (`5xl`).
- **`px-6 pt-20 md:pt-32`**: Внутренние отступы (padding). `x` — по горизонтали (слева и справа), `t` — сверху. На телефонах отступ сверху 20, на ПК — 32, чтобы текст не прилипал к верхнему краю на большом баннере.

### 3. Типографика и UX (удобство чтения)

Обратите внимание на классы у текста:

- **`max-w-4xl` и `max-w-3xl`**: Ограничивает максимальную ширину текстового блока.
  - _Теория UX:_ Если строка текста слишком длинная (например, на весь 4K монитор), глазу пользователя трудно перескакивать с конца одной строки на начало следующей. Ограничение ширины делает текст удобным для чтения.
- **`leading-tight` и `leading-relaxed`**: Это `line-height` (межстрочный интервал).
  - `tight` (плотный) — для заголовка, чтобы буквы не разваливались.
  - `relaxed` (свободный) — для основного текста, чтобы строки не слипались.

### 4. Vue-специфика: Кнопка-ссылка

```html
<router-link to="/catalog" class="inline-block ...">Каталог</router-link>
```

- **Почему `<router-link>`, а не `<button>`?** Потому что это переход на другую страницу (в каталог). С точки зрения семантики HTML, это ссылка. Но визуально мы хотим, чтобы она выглядела как кнопка.
- **`inline-block`**: По умолчанию тег `<a>` (который генерирует `router-link`) является строчным (`inline`). К строчным элементам нельзя применить вертикальные отступы (`padding-top/bottom`, `margin`). `inline-block` позволяет элементу оставаться в строке с текстом, но вести себя как блок (принимать любые отступы и размеры).

### Шпаргалка для ответа на экзамене

**(код, верстка, Vue):**

- **В:** «Как вы сделали так, чтобы текст был поверх картинки?»
  **О:** «Родителю задал `relative`, картинке — `absolute` с растягиванием через `inset-0`. Чтобы текст оказался выше, я использовал `z-index` (`z-10` для текста и `z-0` для картинки).»
- **В:** «Что значит `md:h-[900px]` и почему в скобках?»
  **О:** «Это адаптивность в Tailwind. Префикс `md:` применяет стиль только на экранах шире 768px. Квадратные скобки используются, когда нужно задать точное значение (900px), которого нет в стандартной сетке Tailwind.»
- **В:** «Зачем у ссылки-кнопки класс `inline-block`?»
  **О:** «Ссылка по умолчанию строчный элемент, к ней нельзя применить вертикальные padding и margin. `inline-block` позволяет задать ей размеры и отступы, сохраняя при этом строчное поведение в потоке.»

**(теория HTML/CSS):**

- **В:** «В чем разница между `position: absolute` и `position: fixed`?»
  **О:** «`absolute` позиционирует элемент относительно ближайшего родителя с `position: relative` (или относительно окна браузера, если такого родителя нет). При скролле страницы такой элемент уезжает вместе со страницей. `fixed` позиционирует элемент относительно окна браузера (viewport), и при скролле он остается на месте (как кнопка "Наверх" или шапка сайта).»
- **В:** «Что такое Z-index и как он работает?»
  **О:** «Это CSS-свойство, которое управляет порядком наложения элементов по оси Z (перпендикулярно экрану). Оно работает только для позиционированных элементов (у которых `position` не `static`). Чем больше значение, тем выше элемент в стопке.»
- **В:** «Почему вы ограничили ширину текста через `max-width`?»
  **О:** «Это правило типографики и UX. Слишком длинные строки трудно читать, глаз устает бегать от края до края. Ограничение ширины (обычно 60-80 символов в строке) повышает читабельность.»
