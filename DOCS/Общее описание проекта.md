## Архитектура проекта

index.html (точка входа)
↓ загружает
main.js (запускает Vue)
↓ монтирует
App.vue (корневой компонент: Header + router-view + Footer)
↓ роутер показывает одну из страниц:
├── Home.vue (главная: Hero + О нас + Новинки + Популярное + Отзывы + Контакты)
├── Catalog.vue (каталог: фильтры + товары)
├── ProductPage.vue (страница товара: информация + отзывы)
└── Cart.vue (корзина)
↓ компоненты запрашивают данные через
api/ (сервисный слой: client.js + products.js + categories.js)
↓ Axios делает HTTP-запросы
vite.config.js proxy → https://sites.creatrix-digital.ru/sports-nutrition-store/api/v1
↓ Laravel возвращает JSON
Компоненты отображают данные

## Ключевые технологии

| Технология          | Зачем                                                |
| ------------------- | ---------------------------------------------------- |
| **Vue 3**           | Фреймворк для UI. Composition API (`<script setup>`) |
| **Vue Router**      | Переключение страниц без перезагрузки (SPA)          |
| **Axios**           | HTTP-запросы к API (лучше чем fetch)                 |
| **Tailwind CSS v4** | Утилитарные CSS-классы (не нужно писать свои стили)  |
| **Swiper**          | Слайдер для отзывов                                  |
| **localStorage**    | Хранение корзины в браузере                          |

## 1. `main.js` - точка входа

```JS
import "./assets/main.css";
import { createApp } from "vue";
import App from "./App.vue";
import router from "./router/router.js";

createApp(App).use(router).mount("#app");
```

### Что здесь происходит

**`createApp(App)`** — создаёт экземпляр Vue-приложения. `App` — это корневой компонент, от него начинается всё дерево компонентов.

**`.use(router)`** — устанавливает плагин. Vue Router добавляет глобальные компоненты `<router-view>` (куда рендерится текущая страница) и `<router-link>` (ссылки без перезагрузки).

**`.mount("#app")`** — вставляет приложение в DOM. Ищет элемент с `id="app"` в `index.html` и заменяет его содержимое на отрендеренный Vue.

**`import`** — ES-модули. Файл, который импортируется, выполняется один раз, его экспорт кэшируется.

## 2. `App.vue` - корневой компонент

```Vue
<template>
  <div class="flex flex-col min-h-screen">
    <Header />
    <main class="grow">
      <router-view />
    </main>
    <Footer />
  </div>
</template>

<script setup>
import Header from "./components/Header.vue";
import Footer from "./components/Footer.vue";
</script>
```

### Теория

**`<script setup>`** — синтаксический сахар Vue 3. Всё, что объявлено в этом блоке, автоматически доступно в шаблоне. Не нужно писать `export default { setup() { ... } }`.

**`<router-view />`** — компонент Vue Router. Показывает компонент, соответствующий текущему URL:

- `/` → `Home.vue`
- `/catalog` → `Catalog.vue`
- `/products/1` → `ProductPage.vue`
- `/cart` → `Cart.vue`

**`class="flex flex-col min-h-screen"`** — Tailwind:

- `flex` → `display: flex`
- `flex-col` → `flex-direction: column` (элементы сверху вниз)
- `min-h-screen` → `min-height: 100vh` (минимальная высота = высота экрана)

**`class="grow"`** → `flex-grow: 1` — main растягивается, чтобы Footer всегда был внизу, даже если контента мало.

### Почему Footer всегда внизу?

Без `flex flex-col min-h-screen` + `grow` на main, Footer прилипал бы к концу контента. Если страница короткая — Footer висел бы посередине экрана. С этой конструкцией Footer всегда прижат к низу viewport.

## 3. `vite.config.js` - конфиг сборщика

```JS
import { fileURLToPath, URL } from "node:url";
import { defineConfig } from "vite";
import tailwindcss from "@tailwindcss/vite";
import vue from "@vitejs/plugin-vue";
import vueDevTools from "vite-plugin-vue-devtools";

export default defineConfig({
  plugins: [tailwindcss(), vue(), vueDevTools()],
  resolve: {
    alias: {
      "@": fileURLToPath(new URL("./src", import.meta.url)),
    },
  },

  server: {
    proxy: {
      "/sports-nutrition-store": {
        target: "https://sites.creatrix-digital.ru",
        changeOrigin: true,
        secure: false,
      },
    },
  },
});
```

### Теория

**`defineConfig()`** — функция из Vite, даёт подсказки типов в IDE. Можно было написать объект напрямую, но так удобнее.

**Плагины:**

| Плагин          | Что делает                                              |
| --------------- | ------------------------------------------------------- |
| `vue()`         | Компилирует `.vue` файлы (SFC — Single File Components) |
| `tailwindcss()` | Интегрирует Tailwind v4 с Vite                          |
| `vueDevTools()` | Отладка Vue в браузере (расширение Chrome)              |

**Alias `@` → `./src`:**

```JS
// Без alias:
import { getProducts } from "../../api/products.js";

// С alias:
import { getProducts } from "@/api/products.js";
```

`@` — это сокращение для пути `src/`. Удобно, не нужно считать `../`.

**Proxy!**
Браузеры блокируют запросы с `localhost:5173` на `sites.creatrix-digital.ru` из-за **CORS** (Cross-Origin Resource Sharing). Это политика безопасности.

**Как работает proxy:**

1. Код делает запрос: `GET /sports-nutrition-store/api/v1/products`
2. Браузер думает, что запрос идёт на `localhost:5173` (тот же origin) → CORS не срабатывает
3. Vite видит префикс `/sports-nutrition-store` и перенаправляет на `https://sites.creatrix-digital.ru/sports-nutrition-store/api/v1/products`
4. Сервер отвечает, Vite передаёт ответ браузеру

**Параметры proxy:**

- `target` — куда перенаправлять
- `changeOrigin: true` — меняет заголовок `Host` на целевой домен (некоторые серверы проверяют)
- `secure: false` — разрешает самоподписанные SSL-сертификаты

## 4. `main.css` - стили

```CSS
@import "tailwindcss";
@import "./fonts.css";

html {
  scroll-behavior: smooth;
}
```

### Теория

**`@import "tailwindcss"`** — в Tailwind v4 вся конфигурация через CSS. Раньше (v3) был `tailwind.config.js`, теперь всё в CSS через `@theme`.

**`scroll-behavior: smooth`** — когда кликаем на якорь (`#about`), страница плавно скроллится. Работает вместе с `scrollBehavior` в роутере.

## 5. `router.js` - маршрутизация

```JS
import { createRouter, createWebHistory } from "vue-router";
import Home from "../pages/Home.vue";

const routes = [
  {
    path: "/",
    name: "Home",
    component: Home,
  },
  {
    path: "/catalog",
    name: "Catalog",
    component: () => import("../pages/Catalog.vue"),
  },
  {
    path: "/products/:id",
    name: "Product",
    component: () => import("../pages/ProductPage.vue"),
  },
  {
    path: "/cart",
    name: "Cart",
    component: () => import("../pages/Cart.vue"),
  },
];

const router = createRouter({
  history: createWebHistory(),
  routes,
  scrollBehavior(to, from, savedPosition) {
    if (to.hash) {
      return {
        el: to.hash,
        behavior: "smooth",
      };
    }
    return { top: 0 };
  },
});

export default router;
```

### Теория

**`createWebHistory()`** — использует HTML5 History API. URL выглядят как обычные: `/catalog`, а не `/#/catalog` (hash mode).

**Ленивая загрузка `() => import(...)`:**

```JS
// Обычный импорт — загружается сразу
import Catalog from "../pages/Catalog.vue";

// Ленивый — загружается только когда пользователь переходит на /catalog
component: () => import("../pages/Catalog.vue");
```

То есть каждая страница в отдельном JS-файле. Главная загружается сразу, каталог - только когда нужен.

**`:id` в path** — динамический сегмент. `route.params.id` даёт значение:

- `/products/5` → `id = "5"`
- `/products/12` → `id = "12"`

**`scrollBehavior`** — вызывается при каждом переходе:

- `to.hash` — если URL содержит `#about`, скроллим к этому элементу
- `savedPosition` — если нажали "назад" в браузере, восстанавливаем позицию скролла
- иначе — наверх страницы

## 6. `ProductCard.vue` - карточка товара

### Теория

**`<router-link>`** — компонент Vue Router для навигации. Рендерится как `<a href="...">`, но без перезагрузки страницы.

**`:to="..."`** — динамический проп. `` `/products/${product.id}` `` — шаблонная строка JS, подставляет ID товара.

**`:src` и `:alt`** — сокращение от `v-bind:src`. Привязывает атрибут к значению JS-выражения.

**`|| '/imgs/placeholder-product.png'`** — если `product.image` false (null, undefined, пустая строка), используется заглушка.

**`@click.prevent.stop`** — модификатор события.
`.stop` вызывает `event.stopPropagation()` — клик по кнопке не всплывает к `router-link`, поэтому перехода на страницу товара не происходит.
`.prevent` вызывает `event.preventDefault()` — отменяет стандартное поведение браузера (например, перезагрузку страницы при отправке формы или переход по обычной ссылке `<a>`).

**`mt-auto`** — `margin-top: auto`. Во flex-колонке это прижимает элемент к низу. Если карточки разной высоты, кнопки "В корзину" всё равно будут на одном уровне.

**`defineProps({ product: Object })`** — объявляет проп `product` типа Object. Родитель передаёт его так: `<ProductCard :product="product" />`.

**`useCart()`** — composable (переиспользуемая логика). Возвращает `{ addToCart, removeFromCart, ... }`. Деструктуризация достаёт нужные функции.

## 7. `CatalogFilters.vue` - фильтры каталога

### Теория

**`<aside>`** — семантический HTML-тег для боковой панели (фильтры, реклама, сайдбар).

**`v-for="category in categories"`** — цикл. Для каждого элемента массива `categories` создаёт `<label>`.

**`:key="category.id"`** — уникальный ключ для Vue. Помогает эффективно обновлять DOM при изменениях массива.

**`v-model="selectedCategory"`** — двусторонняя привязка. Когда пользователь выбирает radio, `selectedCategory` обновляется автоматически.

**`@change="$emit('filter-change', ...)"`** — при изменении отправляет событие родителю. Родитель (Catalog.vue) слушает `@filter-change="updateFilters"`.

**`$emit`** — встроенная переменная в `<script setup>`, альтернатива `defineEmits()`.

**`v-model.number`** — модификатор. Автоматически преобразует ввод в число (иначе было бы строкой `"500"`).

**`|| null`** — если поле пустое, передаём `null` вместо `0` или `""`, чтобы API не фильтровал неправильно.

**`useRoute()`** — хук Vue Router. Даёт доступ к текущему маршруту: `route.params` (динамические сегменты), `route.query` (параметры запроса `?category=1`).

**`Number(route.query.category)`** — `route.query.category` всегда строка (`"1"`), преобразуем в число (`1`).

## 8. `CatalogProducts.vue` - сетка товаров

### Теория по коду

**`watch(() => props.filters, ..., { deep: true })`** — следит за изменениями объекта `filters`. `deep: true` — отслеживает вложенные свойства. Без этого watch сработал бы только при замене всего объекта, а не при изменении `filters.categoryId`.

**`const [field, direction] = sortBy.value.split("_")`** — деструктуризация массива. `"price_asc".split("_")` → `["price", "asc"]` → `field = "price"`, `direction = "asc"`.

**`products.value = [...products.value, ...newProducts]`** — spread operator. Создаёт новый массив, объединяя старый и новые товары (для "Показать ещё").

**`response.data.meta?.last_page`** — optional chaining. Если `meta` undefined, не упадёт с ошибкой, вернёт undefined. `|| 1` — fallback.

## 9. `ProductPage.vue` - страница товара

### Теория

**`useRoute()`** — хук для получения текущего маршрута. `route.params.id` — ID товара из URL `/products/5`.

**`response.data.data || response.data`** — защита от разных форматов ответа. Некоторые API возвращают `{ data: {...} }`, другие `{ data: { data: {...} } }`.
