Разбираем последние два файла, которые завершают картину вашего приложения: **`Home.vue`** и **`ProductPage.vue`**. Это файлы из папки `pages` — они представляют собой полноценные страницы, которые Vue Router подставляет в `<router-view>` внутри `App.vue`.

---

### 1. Home.vue (Главная страница)

Этот файл — классический пример **композиции** (сборки) страницы из готовых блоков.

#### А. Принцип "Конструктора" (Композиция компонентов)

```html
<template>
  <div>
    <HeroSection />
    <div id="about"><AboutSection /></div>
    <div id="new"><NewProducts /></div>
    ...
  </div>
</template>
```

- **Зачем так делать:** Главная страница в интернет-магазине обычно очень длинная. Если писать весь её HTML в одном файле, код будет весить тысячи строк, и в нем невозможно будет разобраться. Поэтому мы разбиваем страницу на смысловые блоки (Hero, О нас, Новинки) и создаем для каждого отдельный `.vue` файл. А в `Home.vue` мы просто "складываем" их друг за другом, как блоки Lego.
- **Импорты:** В `<script setup>` мы импортируем каждый компонент, чтобы Vue знал, что означает тег `<HeroSection />`.

#### Б. Якорные ссылки и `id`

```html
<div id="about">
  <AboutSection />
</div>
```

- **Зачем нужны `id`:** Вы оборачиваете секции в `div` с уникальными идентификаторами (`about`, `new`, `popular`). Это нужно для **якорной навигации**.
- **Как это работает:** Если в хедере или футере есть ссылка `<a href="#about">О нас</a>`, браузер при клике на неё попытается прокрутить страницу так, чтобы блок с `id="about"` оказался вверху экрана.
- **Связь с Vue Router:** Помните, в `router.js` мы разбирали функцию `scrollBehavior`? Она перехватывает такие переходы и делает прокрутку **плавной** (`behavior: "smooth"`), вместо резкого прыжка, как в обычном HTML.

### 2. ProductPage.vue (Страница товара)

Этот файл отвечает за отображение конкретного товара (например, `/products/15`). Здесь мы видим, как страница получает данные из адресной строки и передает их компонентам.

#### А. Чтение параметров URL (`useRoute`)

```javascript
import { useRoute } from "vue-router";
const route = useRoute();
const productId = route.params.id;
```

- **`useRoute` vs `useRouter`:** Это частая путаница у новичков.
  - `useRouter` — это "руль". С его помощью мы _переходим_ по страницам (`router.push('/catalog')`).
  - `useRoute` — это "приборная панель". С его помощью мы _читаем_ информацию о текущей странице (какой сейчас URL, какие параметры, какой query).
- **`route.params.id`:** В `router.js` маршрут задан как `/products/:id`. Двоеточие означает, что `id` — это переменная часть пути. Если пользователь зашел на `/products/42`, то `route.params.id` будет равен строке `"42"`.

#### Б. Передача данных через пропсы (Props)

```html
<ProductInfo :product-id="productId" />
<ProductReviews :product-id="productId" />
```

- **`kebab-case` vs `camelCase`:** Обратите внимание на важный нюанс. В JavaScript (в `script`) переменная называется `productId` (camelCase — слитно, с большой буквы). Но в HTML-шаблоне (в `template`) мы пишем `:product-id` (kebab-case — через дефис, маленькими буквами).
- **Почему:** HTML нечувствителен к регистру. Vue автоматически преобразует `product-id` из шаблона в `productId` внутри компонента. Это стандартное соглашение во Vue.
- **Архитектура:** Страница (`ProductPage`) выступает "умным" контейнером. Она знает только одно: _какой ID товара нужно показать_. А вот _как именно_ загрузить и отрисовать товар, знают дочерние компоненты (`ProductInfo` и `ProductReviews`). Они получают ID через пропс и сами делают запрос к API (мы это разбирали в `ProductInfo.vue`).

### Шпаргалка для ответа на экзамене

**(Vue, архитектура, код):**

- **В:** «Почему вы разбили главную страницу на кучу мелких файлов, а не написали всё в `Home.vue`?»
  **О:** «Это принцип композиции и переиспользуемости кода. Так код легче читать, поддерживать и тестировать. Если мне нужно будет изменить блок "Отзывы", я пойду в `ReviewsSection.vue`, а не буду искать его в простыне на 2000 строк в `Home.vue`. К тому же, блок `ReviewsSection` я потом смог переиспользовать (или адаптировать) для других целей.»
- **В:** «Как вы получаете ID товара на странице товара?»
  **О:** «Я использую хук `useRoute` из Vue Router. В маршруте (`router.js`) у меня задан динамический сегмент `/products/:id`. Я обращаюсь к `route.params.id`, получаю ID из URL и передаю его через пропсы в компоненты `ProductInfo` и `ProductReviews`, которые уже сами делают запрос к API.»
- **В:** «Зачем вы оборачиваете секции в `div` с `id` на главной странице?»
  **О:** «Для якорной навигации. Чтобы при клике на пункт меню "О нас" происходил скролл к нужной секции. А в `router.js` я настроил `scrollBehavior`, чтобы этот скролл был плавным.»

**(Теория, Веб, JS):**

- **В:** «Что такое SPA и как в нем работает навигация с якорями (#)?»
  **О:** «В классическом MPA (Multi Page Application) клик по якорю просто прокручивает страницу браузер. В SPA (Single Page Application), если мы переходим по якорю с другой страницы, нам нужно не только прокрутить экран, но и сначала отрисовать нужную страницу. Vue Router решает это через функцию `scrollBehavior`, которая срабатывает после того, как компонент страницы уже смонтирован в DOM.»
- **В:** «В чем разница между `useRouter` и `useRoute`?»
  **О:** «`useRouter` возвращает экземпляр роутера для программного управления навигацией (например, сделать переход `push` или `back`). `useRoute` возвращает реактивный объект текущего маршрута, из которого мы можем читать параметры пути (`params`), query-параметры (`query`) и имя текущего роута.»
- **В:** «Что такое camelCase и kebab-case и почему во Vue они смешиваются?»
  **О:** «camelCase (например, `productId`) — стандарт именования в JavaScript. kebab-case (например, `product-id`) — стандарт в HTML. Поскольку HTML-атрибуты нечувствительны к регистру, Vue использует kebab-case в шаблонах (`<template>`), но автоматически преобразует их в camelCase при передаче в JavaScript-код компонента через пропсы.»
