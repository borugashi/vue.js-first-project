Разбираем три важных файла, которые образуют ядро каталога вашего сайта: **`Catalog.vue`**, **`CatalogProducts.vue`** и **`ProductInfo.vue`**. Здесь используется много продвинутой логики Vue: наблюдатели (`watch`), двустороннее связывание (`v-model`), работа с объектами и пагинация.

---

### 1. Catalog.vue (Родительская страница каталога)

Этот файл выступает «дирижером». Он сам не знает, как рисовать фильтры или товары, он просто соединяет компоненты и хранит общее состояние (выбранные фильтры).

#### А. Паттерн «Props down, Events up» (Данные вниз, События наверх)

Это фундаментальное правило архитектуры Vue.

- **Данные вниз (`:filters="filters"`):** Родитель (`Catalog`) передает данные в дочерний компонент (`CatalogProducts`) через пропсы.
- **События наверх (`@filter-change="updateFilters"`):** Дочерний компонент (`CatalogFilters`) не может менять пропсы напрямую. Когда пользователь меняет фильтр, ребенок «испускает» (emits) событие `filter-change` и передает новые данные. Родитель слушает это событие и обновляет свой `filters`.

#### Б. Спред-оператор (`...`) в JavaScript

```javascript
const updateFilters = (newFilters) => {
  filters.value = { ...filters.value, ...newFilters };
};
```

- **Что это:** Три точки `...` — это **spread-оператор** (оператор расширения). Он «распаковывает» свойства объекта.
- **Зачем здесь:** Мы создаем **новый объект**, копируя в него старые фильтры и накладывая сверху новые.
- _Важно для реактивности:_ Во Vue нельзя просто изменить одно свойство внутри реактивного объекта (например, `filters.value.categoryId = 1`), потому что Vue может не заметить изменение вложенных свойств. Создавая новый объект через `{ ... }`, мы гарантируем, что Vue увидит замену и обновит все зависимые компоненты.

### 2. CatalogProducts.vue (Список товаров, сортировка и пагинация)

Самый сложный компонент с точки зрения логики. Он получает фильтры, следит за их изменением, делает запросы и умеет подгружать товары порциями.

#### А. Двустороннее связывание (`v-model`)

```html
<select v-model="sortBy" class="..."></select>
```

- **Что такое `v-model`:** Это директива Vue, которая объединяет `v-bind` (передача значения) и `v-on:input` (слушание изменений).
- **Как работает:** Когда вы выбираете пункт в `<select>`, переменная `sortBy` в `<script>` автоматически обновляется. И наоборот: если вы измените `sortBy` в коде, `<select>` на экране сразу переключится.

#### Б. Наблюдатель `watch` и опция `deep`

```javascript
watch(
  () => props.filters,
  () => {
    currentPage.value = 1;
    loadProducts();
  },
  { deep: true },
);
```

- **Что такое `watch`:** Хук, который следит за изменением переменной. Как только переменная меняется, выполняется функция-колбэк.
- **Зачем `() => props.filters`:** Мы передаем функцию, которая возвращает объект фильтров.
- **Опция `deep: true`:** По умолчанию `watch` следит только за заменой самого объекта (если бы мы сделали `filters.value = {...}`). Но в нашем случае мы меняем _внутренние свойства_ объекта (например, `priceMin`). `deep: true` заставляет Vue следить за всеми вложенными свойствами объекта и запускать функцию при любом их изменении.
- **Логика:** Если пользователь поменял фильтр, мы обязательно сбрасываем пагинацию на 1-ю страницу (`currentPage.value = 1`) и загружаем товары заново.

#### В. Деструктуризация и разделение строки

```javascript
const [field, direction] = sortBy.value.split("_");
```

- **`split("_")`:** Метод строки в JS, который разрезает строку по разделителю и возвращает массив. Например, `"price_asc".split("_")` вернет `["price", "asc"]`.
- **Деструктуризация массива:** `const [a, b] = ...` — это способ сразу распаковать элементы массива в отдельные переменные. `field` станет `"price"`, `direction` — `"asc"`. Это нужно, чтобы передать бэкенду два отдельных параметра: по какому полю сортировать и в каком направлении.

#### Г. Опциональная цепочка (`?.`)

```javascript
totalPages.value = response.data.meta?.last_page || 1;
```

- **Что это:** Знак вопроса `?.` перед точкой. Если объект слева от него (`response.data.meta`) равен `null` или `undefined`, JS не выдаст ошибку, а сразу вернет `undefined`.
- **Зачем:** Бэкенд может не вернуть объект `meta` (например, если товаров вообще нет). Без `?.` код бы упал с ошибкой `Cannot read properties of undefined`.

#### Д. Пагинация и накопление массива

```javascript
if (append) {
  products.value = [...products.value, ...newProducts];
} else {
  products.value = newProducts;
}
```

- Когда мы меняем фильтр, мы хотим **заменить** товары (`append = false`).
- Когда мы нажимаем «Показать ещё», мы хотим **добавить** новые товары в конец старого списка. Мы берем старый массив, распаковываем его, и следом распаковываем новый массив.

### 3. ProductInfo.vue (Карточка товара)

Компонент, который показывает детали одного товара.

#### А. Валидация типов пропсов

```javascript
const props = defineProps({
  productId: [String, Number],
});
```

- **Массив типов:** Мы указываем, что `productId` может быть либо строкой, либо числом. Это часто бывает, потому что из URL-адреса (`/products/15`) параметр часто приходит как строка, а из кода его можно передать как число.

#### Б. CSS Grid для макета

```html
<div class="grid grid-cols-1 lg:grid-cols-2 gap-12 mb-16"></div>
```

- **`grid-cols-1 lg:grid-cols-2`:** На мобильных устройствах блоки идут друг под другом (1 колонка). На больших экранах (`lg`, от 1024px) экран делится ровно пополам (2 колонки). Это классический макет для страницы товара: слева галерея, справа описание.

#### В. Хлебные крошки

```html
<span class="text-sm text-gray-500">Главная / Каталог</span>
```

- **Что это:** Навигационная цепочка, показывающая пользователю, где он находится.
- **Зачем нужно:** Улучшает UX (пользователь понимает структуру сайта) и SEO (поисковые роботы лучше индексируют вложенность страниц).

### Шпаргалка для ответа на экзамене

**(Vue, код, логика):**

- **В:** «Как вы передаёте данные между фильтрами и списком товаров?»
  **О:** «Родительский компонент `Catalog` хранит состояние фильтров в `ref`. В компонент фильтров он передаёт событие `@filter-change`, а в список товаров — сами фильтры через пропс `:filters`. Когда фильтр меняется, он эмитит событие, родитель обновляет объект, и список товаров автоматически получает новые пропсы.»
- **В:** «Зачем в `watch` вы используете `{ deep: true }`?»
  **О:** «Потому что `filters` — это объект. По умолчанию Vue следит только за заменой ссылки на объект. Если мы меняем вложенное свойство (например, `priceMin`), обычный `watch` это не заметит. `deep: true` включает глубокое наблюдение за всеми вложенными свойствами.»
- **В:** «Как работает кнопка "Показать ещё"?»
  **О:** «Она вызывает функцию `loadMore`, которая увеличивает счётчик страницы (`currentPage++`) и вызывает загрузку с аргументом `append = true`. В итоге новые товары не заменяют старые, а добавляются в конец массива через спред-оператор: `[...old, ...new]`.»
- **В:** «Что делает `v-model` в теге `<select>`?»
  **О:** «Это двустороннее связывание. Оно автоматически синхронизирует выбранное значение в HTML-элементе с переменной `sortBy` в JavaScript. При выборе нового пункта переменная обновляется, срабатывает `watch`, и товары перезагружаются.»

**(Теория JS, HTTP, Архитектура):**

- **В:** «Что такое спред-оператор (`...`) и зачем вы его использовали в `updateFilters`?»
  **О:** «Спред-оператор распаковывает свойства объекта. В `updateFilters` я использую его, чтобы создать **новый** объект, объединив старые и новые фильтры. Это важно для реактивности Vue: фреймворк лучше отслеживает изменения, когда ссылка на объект меняется, а не когда мутируют его внутренние свойства.»
- **В:** «Что такое опциональная цепочка (`?.`) в JS?»
  **О:** «Это безопасный способ доступа к вложенным свойствам объекта. Если объект до точки вопроса равен `null` или `undefined`, выражение сразу вернёт `undefined` без выброса ошибки `TypeError`. Это спасает от краха приложения, если бэкенд вернул неполную структуру данных.»
- **В:** «Как вы разбираете строку сортировки `"price_asc"`?»
  **О:** «Я использую метод строки `split("_")`, который возвращает массив `["price", "asc"]`, и деструктуризацию массива, чтобы сразу присвоить эти значения в переменные `field` и `direction`. Затем я передаю их на бэкенд как отдельные GET-параметры.»
- **В:** «Почему вы сбрасываете `currentPage` на 1 при изменении фильтров или сортировки?»
  **О:** «Это логика пагинации. Если пользователь был на 5-й странице и применил фильтр, по которому товаров всего на 2 страницы, он увидит пустой экран. Поэтому при любом изменении критериев поиска мы всегда должны возвращаться на первую страницу результатов.»
